## 1.1 基本構造パーツ

まずはWebサイトの全体像を理解しましょう。どのサイトを見ても、基本的には同じ構造でできています。

```
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃ ヘッダー（Header）                 ┃
┃ [ロゴ] [ホーム] [サービス] [お問合せ]┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
┏━━━━━━━━━┓ ┏━━━━━━━━━━━━━━━━━━━┓
┃サイドバー┃ ┃  メインコンテンツ    ┃
┃         ┃ ┃                    ┃
┃・関連記事┃ ┃  ここに記事や      ┃
┃・広告    ┃ ┃  商品情報などが    ┃
┃・SNS    ┃ ┃  表示される        ┃
┗━━━━━━━━━┛ ┗━━━━━━━━━━━━━━━━━━━┛
┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃ フッター（Footer）                 ┃
┃ © 2025 会社名 | プライバシーポリシー ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
```

### ヘッダー（Header）

**簡単に言うと**: サイトの一番上にある「看板」のようなもの

**役割**:
- サイトの顔として、どこのサイトか一目でわかるようにする
- 主要ページへのリンクをまとめて配置
- どのページを見ていても、常に表示される共通エリア

**含まれるもの**:
- **ロゴ**: 会社やサービスのロゴ（クリックするとトップページに戻るのが一般的）
- **メインナビゲーション**: 「ホーム」「サービス」「お問い合わせ」などの主要リンク
- **検索窓**: サイト内検索ができる入力欄
- **ログインボタン**: 会員サイトの場合に表示

**実務での会話例**:
- 「ヘッダーのロゴを新しいものに差し替えてください」
- 「ヘッダーのナビに『採用情報』のリンクを追加したいです」
- 「スマホで見たときに、ヘッダーがハンバーガーメニューになるようにしてください」

**よくある質問**:
- Q: ヘッダーは全ページ共通ですか？
- A: はい。基本的にサイト全体で同じヘッダーを使います。これにより、ユーザーがどのページにいても迷わず操作できます。

### フッター（Footer）

**簡単に言うと**: サイトの一番下にある「連絡先や細かい情報」をまとめた場所

**役割**:
- 法的に必要な情報（著作権、プライバシーポリシーなど）を配置
- ヘッダーに入りきらない補助的なリンクを配置
- サイト全体の補足情報を提供

**含まれるもの**:
- **著作権表記**: 「© 2025 会社名」のような表記（必須）
- **プライバシーポリシー**: 個人情報の取り扱いについて（法的に必須）
- **利用規約**: サービスの利用条件
- **お問い合わせリンク**: 連絡先ページへのリンク
- **SNSアイコン**: Twitter、Instagram、FacebookなどのSNSリンク
- **サイトマップ**: サイト全体の構造を示すリンク集

**実務での会話例**:
- 「フッターに会社の住所と電話番号を追加してください」
- 「フッターのSNSアイコンをもっと目立つ色にしたいです」
- 「プライバシーポリシーのリンクがフッターにないと法的にまずいです」

**よくある質問**:
- Q: フッターは必ず必要ですか？
- A: はい。特に企業サイトの場合、著作権表記やプライバシーポリシーは法的に必須です。

### ナビゲーション（Navigation/Nav）

**簡単に言うと**: サイト内の「地図」や「道しるべ」

**役割**:
- ユーザーが目的のページにすぐたどり着けるようにする
- サイト内のどこにいるか、どこに行けるかを明確にする
- サイトの迷子を防ぐ

**種類**:

1. **グローバルナビゲーション（グローバルナビ）**
   - サイト全体で共通の主要メニュー
   - ヘッダー内に配置されることが多い
   - 例: 「ホーム」「サービス」「料金」「お問い合わせ」

2. **ローカルナビゲーション（ローカルナビ）**
   - 特定のセクション内だけで使うメニュー
   - サイドバーに配置されることが多い
   - 例: ブログカテゴリ一覧、商品カテゴリなど

3. **パンくずナビゲーション（パンくずリスト）**
   - 現在地を階層で示すナビ
   - 例: `トップ > スポーツ > サッカー > Jリーグ`

**実務での会話例**:
- 「グローバルナビに『よくある質問』を追加してください」
- 「スマホ表示のときは、ナビをハンバーガーメニュー（三本線のアイコン）にしてください」
- 「パンくずナビがないと、ユーザーがどこにいるかわからなくなります」

**よくある失敗例**:
- メニュー項目が多すぎて、何が重要かわからない（5〜7項目が理想）
- メニューの名前が抽象的で、クリックしても何が出るかわからない

### サイドバー（Sidebar）

**簡単に言うと**: メインコンテンツの横に配置される「おすすめ情報」エリア

**役割**:
- メインコンテンツを邪魔せず、補足情報を提供
- 関連情報へのリンクで回遊率を上げる
- 広告枠として収益化

**含まれるもの**:
- **関連記事**: 今読んでいる記事に関連する他の記事
- **カテゴリ一覧**: ブログのカテゴリリンク
- **人気記事ランキング**: よく読まれている記事
- **広告枠**: バナー広告などの収益化エリア
- **SNSボタン**: フォローボタンやシェアボタン
- **検索窓**: サイト内検索

**実務での会話例**:
- 「サイドバーに人気記事ランキングを表示してください」
- 「スマホ表示のときは、サイドバーをメインコンテンツの下に移動してください」
- 「サイドバーの広告枠のサイズを変更したいです」

**よくある質問**:
- Q: サイドバーは必ず必要ですか？
- A: いいえ。最近はスマホ対応のため、サイドバーなしのシンプルなデザインも増えています。

**スマホ対応の注意点**:
- スマホでは画面幅が狭いため、サイドバーは通常メインコンテンツの下に配置されます
- デザイン時は必ず「PC表示」と「スマホ表示」の両方を考える必要があります

## 1.2 コンテンツパーツ

次に、サイト内のコンテンツを構成する重要なパーツを学びましょう。

### CTAボタン（Call To Action）

**簡単に言うと**: ユーザーに「これをクリックしてほしい！」と促す目立つボタン

**役割**:
- ユーザーに具体的な行動を促す
- サイトの目的（購入、申込、問い合わせなど）を達成させる
- コンバージョン（成果）に直結する最重要パーツ

**具体例**:

```
┌─────────────────────┐
│  今すぐ無料で試す   │  ← サービス紹介ページに設置
└─────────────────────┘

┌─────────────────────┐
│  資料をダウンロード  │  ← BtoB企業サイトでよく使用
└─────────────────────┘

┌─────────────────────┐
│  カートに入れる     │  ← ECサイトの商品ページ
└─────────────────────┘
```

**良いCTAボタンの条件**:
1. **目立つ色**: 周りと違う色で注目を集める（赤、緑、オレンジが多い）
2. **具体的な文言**: 「クリック」ではなく「無料で試す」など具体的に
3. **配置**: ページの重要な場所に配置（ファーストビュー、コンテンツ後など）
4. **サイズ**: タップしやすい大きさ（スマホ対応重要）

**実務での会話例**:
- 「このページのCTAボタンのクリック率が低いので、色を赤に変更してみましょう」
- 「CTAボタンの文言を『詳しく見る』から『無料で資料請求』に変更してください」
- 「スマホでCTAボタンが小さすぎて押しにくいので、もっと大きくしてください」

**よくある失敗例**:
- ボタンが小さすぎて気づかれない
- 文言が抽象的で何が起きるかわからない（「こちら」「クリック」など）
- 色が地味で目立たない
- 1ページに多数のCTAがあり、何を押せばいいかわからない

**計測の重要性**:
- CTAボタンのクリック率は必ず計測する
- A/Bテストで色や文言を変えて効果を比較する
- クリック率2〜5%が一般的な目安

### フォーム（Form）

**簡単に言うと**: ユーザーが情報を入力して送信する「入力欄」の集まり

**役割**:
- ユーザーから情報（名前、メールアドレス、質問内容など）を受け取る
- お問い合わせ、会員登録、アンケートなど様々な用途に使用
- ビジネスにおける「リード獲得」の最重要ポイント

**主な種類**:

1. **お問い合わせフォーム**
```
お名前:     [________________]
メール:     [________________]
お問い合わせ内容:
           [________________]
           [________________]
           [________________]
           [  送信する  ]
```

2. **会員登録フォーム**
```
メールアドレス: [________________]
パスワード:     [________________]
パスワード確認: [________________]
           [  登録する  ]
```

3. **アンケートフォーム**
```
満足度: ○ 大変満足 ○ 満足 ○ 普通 ○ 不満
ご意見: [________________]
           [  送信する  ]
```

**フォーム設計の重要原則**:
1. **入力項目は最小限に**: 項目が多いほど離脱率が上がる
2. **必須項目を明示**: どれが必須かを赤い「※」などで示す
3. **エラーメッセージを親切に**: 「入力エラー」ではなく「メールアドレスの形式が正しくありません」
4. **プライバシーポリシーへのリンク**: 個人情報を扱う場合は必須
5. **送信後の画面を用意**: 「送信完了」画面で安心感を与える

**実務での会話例**:
- 「お問い合わせフォームの項目が多すぎて、途中で離脱されています。必須項目を3つに絞りましょう」
- 「フォーム送信後に確認メールが届くように設定してください」
- 「スマホでフォームが入力しづらいので、入力欄を縦並びにしてください」

**よくある失敗例**:
- 必須項目が多すぎて、ユーザーが面倒になって離脱する
- エラーメッセージがわかりにくく、何を直せばいいかわからない
- 送信ボタンを押しても反応がなく、ユーザーが不安になる
- スマホで入力しづらい（文字が小さい、タップしにくいなど）

**計測すべき指標**:
- **フォーム到達率**: フォームのあるページまでたどり着いた人の割合
- **フォーム離脱率**: フォームを開いたが送信せずに離脱した人の割合
- **フォーム完了率**: フォームを正常に送信できた人の割合

### パンくずリスト（Breadcrumb）

**簡単に言うと**: 「今、サイトのどこにいるか」を示す道しるべ

**名前の由来**: 童話『ヘンゼルとグレーテル』で、帰り道がわかるようにパンくずを落としながら歩いた話から

**表示例**:
```
トップ > スポーツ > サッカー > Jリーグ > 浦和レッズ
```

**役割**:
1. **ユーザーへの位置情報提供**: 今どのページにいるか一目でわかる
2. **簡単な階層移動**: クリックすれば上の階層に戻れる
3. **SEO効果**: 検索エンジンにサイト構造を理解させやすい

**配置場所**:
- ページの上部（ヘッダーとメインコンテンツの間）
- ページの最上部より少し下（ヘッダーの直下）

**実務での会話例**:
- 「パンくずリストがないと、ユーザーが今どのカテゴリを見ているかわからなくなります」
- 「パンくずリストをクリックできるようにして、簡単に上の階層に戻れるようにしてください」
- 「SEO対策として、パンくずリストを構造化データでマークアップしてください」

**よくある質問**:
- Q: パンくずリストは必ず必要ですか？
- A: 階層が深いサイト（ECサイト、大規模メディアなど）では必須です。1〜2階層の小規模サイトでは省略されることもあります。

**SEO効果**:
- 検索結果にパンくずリストが表示されることがある
- サイト構造が明確になり、検索エンジンがページを理解しやすくなる
- 内部リンクが増え、サイト全体のSEO効果が向上

**実際の表示イメージ**:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ヘッダー（ロゴ、メニューなど）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
トップ > スポーツ > サッカー > 記事タイトル
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

記事のメインコンテンツがここに表示される
...
```
