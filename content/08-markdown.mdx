## ● なぜ必要か

AIとのやり取り、GitHubでの資料管理、技術者との共通言語として必須。今後のAI駆動時代に認識しやすい形式だから。

## 8.1 基本記法

### 見出し

```markdown
# 大見出し（H1）
## 中見出し（H2）
### 小見出し（H3）
```

### 文字装飾

```markdown
**太字**
*斜体*
~~取り消し線~~
`インラインコード`
```

### リスト

```markdown
- 順序なしリスト1
- 順序なしリスト2
  - 入れ子リスト

1. 順序ありリスト1
2. 順序ありリスト2
```

### リンクと画像

```markdown
[リンクテキスト](URL)
![画像の説明](画像URL)
```

## 8.2 表とコードブロック

### 表

```markdown
| 項目1 | 項目2 | 項目3 |
|-------|-------|-------|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### コードブロック

````markdown
```言語名
コードをここに書く
```
````

## 8.3 実務での活用場面

### 議事録作成

- 見出しで議題を整理
- チェックボックスでアクションアイテム管理
- リンクで参考資料を添付

### 仕様書作成

- 構造的な文書作成
- コードサンプルの記載
- 表での要件整理

## 理解度チェック問題 8-1

1. 中見出しを作るマークダウン記法は？
2. 太字にする記法は？
3. 順序なしリストを作る記法は？
4. リンクを作る記法は？

**解答**: 1. ## 2. **文字** 3. - 4. [テキスト](URL)
