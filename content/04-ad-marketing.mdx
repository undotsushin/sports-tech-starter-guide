## 4.1 動画広告の種類

動画広告は、YouTubeやTVerなどの動画コンテンツの中に表示される広告です。挿入される位置によって効果が大きく変わります。

```
動画広告の配置タイミング

[プリロール]     [ミッドロール]     [ポストロール]
     ↓               ↓                 ↓
┌────────┬───────────────┬────────┐
│ 広告   │  動画本編      │ 広告   │
│ 15秒  │  本編 中に広告  │ 30秒  │
└────────┴───────────────┴────────┘
 再生前      再生途中          再生後
```

### プリロール（Pre-roll）

**簡単に言うと**: 動画が始まる前に流れる広告（YouTubeでよく見るやつ）

**定義**:
- 動画コンテンツの再生前に必ず流れる広告
- 最も一般的な動画広告の形式
- 5秒でスキップできるものと、最後まで見ないといけないものがある

**広告の長さ**:
- **スキッパブル広告**: 5秒経過後にスキップ可能（長さは自由）
- **ノンスキッパブル広告**: 15秒または30秒（スキップ不可）
- **バンパー広告**: 6秒（スキップ不可、短くて印象的）

**メリット**:
1. **高い視聴率**: 動画を見る前なので必ず表示される
2. **認知度向上**: ブランドや商品を多くの人に知ってもらえる
3. **インパクト大**: 動画を見る気満々の状態なので注目度が高い

**デメリット**:
1. **ユーザーに嫌がられる**: 本編を見たいのに邪魔される
2. **スキップされやすい**: 5秒でスキップされることが多い
3. **コストが高い**: 視聴単価が高め

**実務での活用**:
- 新商品の認知度を上げたいとき
- イベントの告知をしたいとき
- ブランドイメージを広めたいとき

**実務での会話例**:
- 「プリロール広告で、最初の5秒でインパクトを出しましょう」
- 「スキップ率が80%なので、最初の5秒を改善しましょう」
- 「バンパー広告なら6秒で確実に見てもらえます」

**料金形態**:
- CPV（Cost Per View）: 視聴1回あたり3円〜20円程度
- 30秒以上視聴（またはクリック）された場合に課金されるのが一般的

### ミッドロール（Mid-roll）

**簡単に言うと**: 動画の途中でいきなり入ってくる広告（テレビCMみたいな）

**定義**:
- 動画コンテンツの途中に挿入される広告
- 10分以上の長尺コンテンツでよく使われる
- YouTubeでは動画が8分以上ないと設定できない

**メリット**:
1. **視聴継続率が高い**: すでに内容に引き込まれているので離脱しにくい
2. **収益性が高い**: 広告単価が高く設定されやすい
3. **確実に表示される**: 飛ばされにくい

**デメリット**:
1. **ユーザー体験を損なう**: 物語や説明の途中で入るので嫌がられる
2. **最も不評**: アンケートで最も嫌われる広告形式
3. **設置場所が難しい**: タイミングを間違えるとクレームになる

**実務での活用**:
- 長い動画コンテンツ（15分以上）での収益化
- すでに引き込まれているユーザーへの訴求
- テレビCMのようなマス向け広告

**実務での会話例**:
- 「ミッドロールは収益が高いですが、ユーザーが離脱する可能性もあります」
- 「盛り上がる直前ではなく、キリの良い場所に入れましょう」

**料金形態**:
- プリロールより10〜30%高い単価が一般的

### ポストロール（Post-roll）

**簡単に言うと**: 動画が終わった後に流れる広告

**定義**:
- 動画コンテンツが終了した後に流れる広告
- 最も視聴率が低いが、興味のある人だけが見る
- 最後まで見てくれた=熱量の高いユーザー

**メリット**:
1. **熱量の高いユーザー**: 最後まで見るほど興味がある人
2. **詳細な説明が可能**: 時間をかけて商品を説明できる
3. **コストが安い**: 視聴率が低い分、単価が安い

**デメリット**:
1. **視聴率が非常に低い**: 80〜90%の人がすぐ閉じる
2. **効果が限定的**: リーチ数を稼げない

**実務での活用**:
- 詳しい商品説明やデモ動画
- 関連動画への誘導
- ブランドストーリーの発信

**実務での会話例**:
- 「ポストロールはコスパは良いですが、リーチは稼げません」
- 「興味のある人だけに詳しく説明したいならポストロールが最適です」

**よくある質問**:
- Q: どの形式が一番効果的ですか？
- A: 目的によります。認知度を上げたいなら「プリロール」、収益を最大化したいなら「ミッドロール」、熱量の高いユーザーに訴求したいなら「ポストロール」です。

## 4.2 広告配信システム

オンライン広告は複雑なシステムで配信されています。仕組みを理解すると、効率的な広告運用ができます。

### アドネットワーク（Ad Network）

**簡単に言うと**: たくさんのWebサイトに一気に広告を出せるサービス

**定義**:
- 複数のWebサイトやアプリをまとめて、広告主に販売するサービス
- 広告主は1つの窓口で多数の媒体に広告を出せる

**代表的なサービス**:
- Google AdSense（最大手）
- Yahoo!広告
- i-mobile
- nend

**仕組み**:
```
  広告主              アドネットワーク         Webサイト
    │                      │                    │
    │  広告を出したい      │                    │
    ├─────────────────────>│                    │
    │                      │  広告枠を提供      │
    │                      │<───────────────────┤
    │                      │                    │
    │  一括で配信          │  複数サイトに配信  │
    │<─────────────────────┤─────────────────────>
```

**メリット**:
1. **一括管理**: 1つの管理画面で複数サイトに配信
2. **手間が少ない**: 個別に交渉する必要がない
3. **スケールメリット**: 大量に出稿すると単価が下がる

**実務での会話例**:
- 「まずはGoogle AdSenseで配信して、反応を見ましょう」
- 「アドネットワークなら、すぐに大量のサイトに配信できます」

### DSP（Demand Side Platform）

**簡単に言うと**: 広告主が使う、賢い広告配信システム

**定義**:
- 広告主側（広告を出したい人）のための広告配信プラットフォーム
- AIが自動で最適な広告枠に入札して配信
- リアルタイムで入札を行う（RTB: Real Time Bidding）

**主な機能**:
1. **ターゲティング**: 年齢、性別、興味関心で絞り込み
2. **自動入札**: AIが最適な価格で自動入札
3. **効果測定**: リアルタイムで成果を確認
4. **予算管理**: 上限を設定して使いすぎを防止

**代表的なDSP**:
- Google Display & Video 360
- Yahoo! DSP
- FreakOut
- MicroAd BLADE

**実務での会話例**:
- 「DSPを使えば、自動で効率的な配信ができます」
- 「ターゲティングを絞りすぎると、配信量が減ります」

### SSP（Supply Side Platform）

**簡単に言うと**: Webサイト運営者が使う、広告収益を最大化するシステム

**定義**:
- 媒体側（広告枠を売りたい人）のための広告販売プラットフォーム
- 複数の広告主から最も高い価格を提示した広告を自動選択
- 広告枠の価値を最大化

**主な機能**:
1. **価格最適化**: リアルタイムで最高値の広告を選択
2. **複数DSPと接続**: 多くの広告主と競わせる
3. **収益レポート**: 詳細な収益分析

**実務での会話例**:
- 「SSPを導入したら、広告収益が30%アップしました」
- 「複数のSSPを併用して、収益を最大化しましょう」

**DSPとSSPの関係**:
```
広告主 → DSP ←→ アドエクスチェンジ ←→ SSP ← Webサイト
       （買う側）    （市場）      （売る側）
```

## 4.3 効果測定指標

広告の効果を正しく測定しないと、お金を無駄にしてしまいます。重要な指標を理解しましょう。

### インプレッション（Impression）

**簡単に言うと**: 広告が表示された回数

**定義**:
- 広告がユーザーの画面に表示された回数
- 英語で「imp」と略されることが多い
- 「リーチ」とも呼ばれる

**例**:
- 10,000インプレッション = 10,000回表示された
- ただし、同じ人が何回も見ている可能性がある

**実務での会話例**:
- 「今月のインプレッションは100万impでした」
- 「インプレッション数は多いのに、クリックが少ないですね」

### クリック率（CTR: Click Through Rate）

**簡単に言うと**: 広告を見た人のうち、何%がクリックしたか

**計算式**:
```
CTR = (クリック数 ÷ インプレッション数) × 100

例: 10,000回表示されて、30回クリックされた
CTR = (30 ÷ 10,000) × 100 = 0.3%
```

**目安**:
- ディスプレイ広告: 0.1〜0.3%
- 検索広告: 2〜5%
- SNS広告: 0.5〜2%

**CTRが低い原因**:
1. 広告のデザインが悪い
2. ターゲティングがずれている
3. 広告文が魅力的でない

**実務での会話例**:
- 「CTRが0.05%しかないので、クリエイティブを変更しましょう」
- 「CTRが1%もあるのに、コンバージョンが少ないのはなぜでしょう？」

### コンバージョン率（CVR: Conversion Rate）

**簡単に言うと**: クリックした人のうち、何%が購入や申込をしたか

**定義**:
- コンバージョン = 目標達成（購入、申込、資料請求など）
- 最も重要な指標

**計算式**:
```
CVR = (コンバージョン数 ÷ クリック数) × 100

例: 100人がクリックして、3人が購入
CVR = (3 ÷ 100) × 100 = 3%
```

**目安**:
- ECサイト: 1〜3%
- BtoB（問い合わせ）: 2〜5%
- アプリダウンロード: 10〜30%

**CVRが低い原因**:
1. ランディングページが悪い
2. 価格が高い
3. フォームが複雑すぎる

**実務での会話例**:
- 「CVRが1%を切っているので、LPを改善しましょう」
- 「A/Bテストの結果、CVRが2倍になりました！」

### CPA（Cost Per Acquisition）

**簡単に言うと**: 1件の成果を獲得するのにかかった費用

**計算式**:
```
CPA = 広告費 ÷ コンバージョン数

例: 100万円使って、50件獲得
CPA = 1,000,000 ÷ 50 = 20,000円
```

**重要性**:
- CPAが目標より低い = 利益が出る
- CPAが目標より高い = 赤字

**目標CPAの決め方**:
```
商品価格: 10,000円
利益率: 30% → 利益 3,000円
目標CPA: 3,000円以下なら黒字
```

**実務での会話例**:
- 「CPAが目標の5,000円を超えているので、広告を停止しましょう」
- 「このキーワードはCPAが低いので、予算を増やしましょう」

### ROAS（Return On Ad Spend）

**簡単に言うと**: 広告費1円に対して、何円の売上があったか

**計算式**:
```
ROAS = (売上 ÷ 広告費) × 100%

例: 100万円使って、300万円売れた
ROAS = (3,000,000 ÷ 1,000,000) × 100 = 300%
```

**目安**:
- ROAS 100%以上 = 広告費回収できた
- ROAS 200%以上 = 利益が出ている（利益率50%の場合）
- ROAS 500%以上 = 非常に優秀

**実務での会話例**:
- 「ROASが150%なので、広告費は回収できていますが利益は出ていません」
- 「このキャンペーンはROAS 400%で大成功です！」
