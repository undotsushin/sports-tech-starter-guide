## 5.1 データベース基礎

### データベースとは

**簡単に言うと**: データを整理して保存する「デジタルの本棚」

**定義**:
- 大量のデータを効率的に保存・検索・管理するシステム
- ExcelやGoogleスプレッドシートの超強力版
- Webサイトやアプリの裏側で動いている

**具体例**:
- ECサイト: 商品情報、在庫、注文履歴を保存
- SNS: ユーザー情報、投稿、いいねを保存
- 銀行: 口座情報、取引履歴を保存

**データベースの種類**:

1. **リレーショナルデータベース（RDB）**
   - データを表（テーブル）形式で管理
   - Excelのような表をイメージ
   - 代表的なもの: MySQL、PostgreSQL、Oracle
   - 用途: 一般的なWebサイト、業務システム

```
ユーザーテーブル
┌────┬────────┬─────────────┬────────┐
│ID  │ 名前   │ メール      │ 年齢   │
├────┼────────┼─────────────┼────────┤
│ 1  │ 田中   │ tanaka@...  │ 25    │
│ 2  │ 佐藤   │ sato@...    │ 30    │
└────┴────────┴─────────────┴────────┘
```

2. **NoSQLデータベース**
   - 表形式にとらわれない柔軟なデータ管理
   - JSON形式でデータを保存
   - 代表的なもの: MongoDB、DynamoDB、Redis
   - 用途: ビッグデータ、リアルタイムアプリ

**実務での会話例**:
- 「ユーザー数が100万人を超えたので、データベースをスケールアップしましょう」
- 「このクエリが遅いので、インデックスを張りましょう」
- 「データベースのバックアップは毎日取っていますか？」

**よくある質問**:
- Q: データベースとExcelの違いは？
- A: Excelは数万行までですが、データベースは数億行でも高速に処理できます。また、同時に何千人がアクセスしても大丈夫です。

### API（Application Programming Interface）

**簡単に言うと**: アプリ同士がデータをやり取りするための「窓口」

**定義**:
- システム間でデータをやり取りする仕組み
- レストランで言うと「メニュー」のようなもの
- メニュー（API）を見て注文（リクエスト）すると、料理（データ）が届く

**APIの仕組み**:

```
   あなたのアプリ          API          外部サービス
        │                  │                │
        │  天気を教えて!    │                │
        ├─────────────────>│                │
        │                  │  天気データ要求 │
        │                  ├───────────────>│
        │                  │                │
        │                  │  天気: 晴れ 25℃│
        │  晴れ 25℃       │<───────────────┤
        │<─────────────────┤                │
```

**代表的なAPI**:

1. **Google Maps API**
   - 地図表示、ルート検索、場所検索
   - 自分のWebサイトに地図を埋め込める
   - 月28,000回まで無料

2. **Twitter API**
   - ツイートの投稿、取得
   - タイムラインの表示
   - 自動投稿botなどに使用

3. **Stripe API**
   - クレジットカード決済
   - 自分のサイトに決済機能を追加

4. **LINE Messaging API**
   - LINEボットの作成
   - メッセージの送受信

**APIのメリット**:
1. **開発時間の短縮**: ゼロから作らなくて済む
2. **コスト削減**: 既存サービスを活用
3. **品質向上**: 専門企業のサービスを利用
4. **保守が楽**: 更新を相手がやってくれる

**実務での会話例**:
- 「決済機能はStripe APIで実装しましょう」
- 「Google Maps APIの無料枠を超えそうなので、料金プランを確認してください」
- 「このAPIはレート制限があるので、秒間100リクエストまでに抑えましょう」

**料金体系**:
- 無料枠あり: Google Maps、Twitter、多くのAPIで一定量まで無料
- 従量課金: 使った分だけ支払い（1,000リクエストあたり◯円など）
- 月額固定: 月額数千円〜数万円

**よくある質問**:
- Q: APIを使うのに許可は必要ですか？
- A: はい。各サービスでアカウント登録とAPI キーの取得が必要です。また、利用規約を守る必要があります。

## 5.2 クラウドサービス

**簡単に言うと**: インターネット上のサーバーを借りて、システムを動かすサービス

**従来の方式との違い**:

```
従来（オンプレミス）        クラウド
┌─────────────────┐    ┌─────────────────┐
│自社でサーバー購入│    │月額で借りる      │
│初期費用: 数百万円│    │初期費用: 0円    │
│設置場所が必要   │    │設置場所不要      │
│故障したら自分で │    │故障時も自動対応  │
└─────────────────┘    └─────────────────┘
```

### 主要クラウドサービス

1. **AWS（Amazon Web Services）**
   - 運営: Amazon
   - シェア: 世界No.1（約32%）
   - 特徴: サービス数が最も豊富（200以上）
   - 代表的なサービス: EC2（サーバー）、S3（ストレージ）、RDS（データベース）
   - 料金: 従量課金、無料枠あり（12ヶ月間）
   - 向いている: 大規模サービス、柔軟性重視

2. **Google Cloud Platform（GCP）**
   - 運営: Google
   - シェア: 世界3位（約11%）
   - 特徴: AI・機械学習、ビッグデータ処理が強い
   - 代表的なサービス: Compute Engine、BigQuery、Cloud AI
   - 料金: AWSより若干安い傾向
   - 向いている: データ分析、AI活用

3. **Microsoft Azure**
   - 運営: Microsoft
   - シェア: 世界2位（約23%）
   - 特徴: Microsoft製品（Windows Server、Office365）との親和性が高い
   - 代表的なサービス: Virtual Machines、Azure SQL Database
   - 料金: AWSと同程度
   - 向いている: 企業システム、Microsoft製品ユーザー

### クラウドのメリット

1. **初期投資不要**
   - サーバー購入不要（数百万円の節約）
   - すぐに始められる（数分で起動）

2. **スケーラビリティ**
   - アクセスが増えたら自動でサーバー増強
   - アクセスが減ったら自動で縮小（コスト削減）

3. **高い可用性**
   - 99.99%の稼働率保証（年間のダウンタイムが53分以下）
   - 世界中にデータセンターがある

4. **自動バックアップ**
   - データが自動で複数箇所に保存される
   - 災害時も安心

5. **セキュリティ**
   - 専門チームが24時間監視
   - 最新のセキュリティ対策が自動適用

**実務での会話例**:
- 「AWSでサーバーを立てて、まずは小さく始めましょう」
- 「アクセスが急増したので、Auto Scalingで自動対応しました」
- 「クラウド費用が月100万円を超えたので、最適化が必要です」

**よくある質問**:
- Q: クラウドの方が絶対に安いですか？
- A: 小規模なら安いですが、大規模になると自社サーバーの方が安い場合もあります。ただし、保守の手間を考えるとクラウドが有利です。

## 5.3 セキュリティ基礎

**簡単に言うと**: 個人情報や重要なデータを守るためのルールと技術

### 個人情報保護

Webサイトやアプリで個人情報を扱う場合、法律を守る必要があります。

**個人情報とは**:
- 名前、住所、電話番号、メールアドレス
- 生年月日、性別
- 顔写真、音声
- クレジットカード情報
- IPアドレス、Cookie情報

**個人情報保護法（日本）**:
- 日本国内で個人情報を扱う全ての事業者が対象
- 5,000人以上の個人情報を扱う場合、届出が必要
- 違反すると: 最大1億円の罰金

**守るべきルール**:
1. **利用目的の明示**: 何のために個人情報を使うか明記
2. **同意の取得**: 本人の同意なしに収集・利用しない
3. **安全管理**: 漏洩しないように適切に管理
4. **第三者提供の制限**: 勝手に他社に渡さない
5. **開示請求への対応**: 本人が「自分の情報を見せて」と言ったら開示

**GDPR（EU一般データ保護規則）**:
- EUの法律だが、日本企業も対象になる場合がある
- EU在住者の個人情報を扱う場合は適用
- 違反すると: 最大2,000万ユーロ（約30億円）または全世界売上の4%

**実務での会話例**:
- 「個人情報を扱うので、プライバシーポリシーを必ず掲載してください」
- 「GDPRに対応するため、Cookie同意バナーを表示しましょう」
- 「個人情報が漏洩したら、すぐに報告義務があります」

**よくある違反例**:
- プライバシーポリシーがない
- 同意なしにメルマガを送る
- パスワードを平文で保存（暗号化せず保存）
- 退会後も個人情報を削除しない

### Cookie規制

**簡単に言うと**: Webサイトがユーザーの行動を追跡する仕組みへの規制

**Cookieとは**:
- Webサイトがブラウザに保存する小さなデータ
- ログイン状態の保持、カートの中身の記憶などに使用
- ユーザーの行動を追跡することも可能

**Cookieの種類**:

1. **ファーストパーティーCookie**
   - 訪問したサイト自身が発行
   - 例: ログイン状態の保持、カート情報
   - 規制: 緩い

2. **サードパーティーCookie**
   - 広告会社など第三者が発行
   - 例: 複数サイトをまたいだ行動追跡、ターゲティング広告
   - 規制: 厳しい（廃止の方向）

**規制の動向**:
- Google Chrome: 2024年にサードパーティーCookie廃止予定
- Safari: 既に厳しく制限
- Firefox: デフォルトでブロック

**実務での影響**:
1. **広告効果測定の精度低下**: どの広告経由で来たかわかりにくくなる
2. **リターゲティング広告が難しい**: 一度サイトに来た人を追跡できない
3. **Cookie同意バナーの必須化**: 「Cookieを使用します」という同意が必要

**対応策**:
- ファーストパーティーCookieに移行
- サーバーサイドでの計測に切り替え
- Googleアナリティクス 4（GA4）への移行

**実務での会話例**:
- 「Cookie規制でリターゲティング広告の効果が落ちています」
- 「Cookie同意バナーを実装して、GDPR対応しましょう」
- 「サードパーティーCookie廃止後の対策を考える必要があります」
