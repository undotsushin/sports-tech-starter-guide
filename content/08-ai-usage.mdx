## 8.1 主要AIツールの特徴と使い分け

**簡単に言うと**: ChatGPT、Claude、Geminiの違いを知り、目的に応じて使い分ける

### ChatGPT（OpenAI）

**簡単に言うと**: 最も有名で汎用性が高いAI

**特徴**:
```
強み
- 最も汎用性が高い
- プラグインが豊富
- 日本語対応が良い
- ユーザー数が最多（1億人以上）

弱み
- 長文処理は苦手（約3,000字まで）
- 最新情報は2023年4月まで（GPT-4の場合）
```

**料金**:
- 無料版: GPT-3.5（制限あり）
- 有料版: 月20ドル（GPT-4、画像生成DALL-E付き）
- API: 従量課金（GPT-4: 1,000トークンあたり0.03ドル）

**得意なタスク**:
```
1. コード生成
   - Python、JavaScript、HTML/CSSなど
   - バグ修正、リファクタリング

2. 文章作成
   - メール、報告書、プレゼン資料
   - キャッチコピー、広告文

3. アイデア出し
   - ブレインストーミング
   - 企画案の発想

4. 翻訳
   - 自然な日英翻訳
   - 専門用語にも対応

5. データ分析
   - Excelデータの分析
   - グラフ作成
```

**実務での使い方**:
```
× 悪い使い方:
「良い企画を考えて」

○ 良い使い方:
「20代女性向けのランニングイベントの企画を5つ考えてください。
条件:
- 予算: 100万円
- 期間: 3ヶ月
- 目標参加者数: 300人
- 開催地: 東京都内
各企画について、以下を記載してください:
1. 企画名
2. 概要（100字）
3. 予算配分
4. 期待効果」
```

### Claude（Anthropic）

**簡単に言うと**: 長文処理と正確性に優れたAI

**特徴**:
```
強み
- 長文処理が得意（約20万字まで）
- 正確性が高い
- 倫理的な回答
- 日本語品質が良い

弱み
- プラグインがない
- 画像生成機能がない
- 知名度がChatGPTより低い
```

**料金**:
- 無料版: Claude 3 Sonnet（制限あり）
- 有料版: 月20ドル（Claude 3 Opus）
- API: 従量課金（1,000トークンあたり0.015ドル）

**得意なタスク**:
```
1. 長文の要約
   - 論文、レポート、書籍の要約
   - 会議の議事録まとめ

2. 長文の分析
   - 契約書のレビュー
   - 提案書の改善提案

3. 文章の校正
   - 表現の改善
   - 誤字脱字チェック

4. コードレビュー
   - セキュリティチェック
   - 改善提案

5. 議論の整理
   - 論点の整理
   - 意見のまとめ
```

**実務での使い方**:
```
長文要約の例:

「以下の提案書（5,000字）を500字で要約してください。
特に以下の点に注目してください:
- 提案の核心
- 予算と期間
- 期待効果
- リスク

【提案書】
...（長文）...」
```

### Gemini（Google）

**簡単に言うと**: Google検索と連携した最新情報に強いAI

**特徴**:
```
強み
- リアルタイム情報に強い
- Google検索と連携
- YouTube動画の要約が得意
- Google Workspace連携

弱み
- 日本語の精度がやや低い
- 長文生成は苦手
- 創造性はChatGPTに劣る
```

**料金**:
- 無料版: Gemini Pro（制限あり）
- 有料版: 月2,900円（Gemini Advanced、Google One AI Premium）
- API: 従量課金

**得意なタスク**:
```
1. 最新情報の検索
   - ニュース、トレンド
   - 株価、為替

2. YouTube動画の要約
   - 長時間動画の要約
   - 重要ポイント抽出

3. Google連携
   - Gmail、ドキュメント、スプレッドシート
   - Google Maps

4. 画像認識
   - 画像の説明
   - OCR（文字認識）

5. 競合調査
   - 最新の競合情報
   - 市場動向
```

**実務での使い方**:
```
最新情報の活用:

「2024年のランニングシューズのトレンドを調べて、
以下の観点でまとめてください:
```


## 8.2 プロンプトエンジニアリング実践

**簡単に言うと**: AIから最高の回答を引き出すための質問テクニック

### 良いプロンプトの6要素

```
1. 役割設定（Role）
   「あなたは○○の専門家です」

2. 具体的指示（Task）
   「××を作成してください」

3. 背景情報（Context）
   「△△という状況で」

4. 制約条件（Constraints）
   「□□以内で」「□□を含めて」

5. 出力形式（Format）
   「リスト形式で」「表で」

6. 例示（Examples）
   「このような形式で」
```

### 悪いプロンプト vs 良いプロンプト

**例1: 資料作成**

```
× 悪いプロンプト:
「資料作って」

問題点:
- 何の資料か不明
- 誰向けか不明
- どんな内容か不明
- 形式が不明

○ 良いプロンプト:
「あなたは経験豊富な営業資料作成の専門家です。

【依頼内容】
スポーツメディア向けの広告提案資料を作成してください。

【背景】
クライアント: 大手スポーツ用品メーカー
商品: 新作ランニングシューズ（定価15,000円）
ターゲット: 20-30代のランニング愛好家（年収400-600万円）
予算: 月500万円
期間: 3ヶ月キャンペーン

【出力形式】
1. 提案概要（100文字以内）
2. ターゲット分析（200文字）
3. 配信プラン（媒体別予算配分を表で）
4. 期待効果（具体的な数値で）
5. スケジュール（ガントチャート形式で）

【制約】
- 全体で1,500文字以内
- 数値は必ず根拠を示す
- 競合との差別化ポイントを含める」
```

**例2: コード生成**

```
× 悪いプロンプト:
「ログイン機能作って」

○ 良いプロンプト:
「あなたは経験豊富なWebエンジニアです。

【依頼内容】
Node.js + Expressでログイン機能を実装してください。

【要件】
- メールアドレスとパスワードでログイン
- パスワードはbcryptで暗号化
- JWTでセッション管理
- ログイン失敗は3回まで（それ以上はロック）
- ログイン履歴をデータベースに保存

【技術スタック】
- Node.js 18.x
- Express 4.x
- MongoDB
- JWT

【出力形式】
1. コード全文（コメント付き）
2. 必要なnpmパッケージ
3. 環境変数の設定例
4. テストコード
5. セキュリティ上の注意点

【制約】
- ES6記法を使用
- エラーハンドリングを必ず含める
- セキュリティベストプラクティスに従う」
```


## 8.5 注意点とベストプラクティス

**やってはいけないこと**

```
× 機密情報を入力
  - 顧客情報
  - 社内の機密データ
  - パスワード

× AIの回答を無批判に使用
  - 必ず人間が確認
  - ファクトチェック

× 著作権侵害
  - AIの回答をそのままコピペ
  - 必ず自分の言葉で

× 倫理的に問題のある使用
  - フェイクニュース作成
  - 詐欺文書作成
```

**ベストプラクティス**

```
○ AI + 人間の役割分担

AI担当:
- 下書き作成
- アイデア出し
- 情報収集
- 校正

人間担当:
- 最終確認
- 創造性
- 判断
- 責任
```

**継続学習の方法**

```
毎日（5分）:
- 新しいプロンプトを試す
- 成功事例を記録

毎週（30分）:
- 成功事例を共有
- 新機能を確認

毎月（2時間）:
- 業務効率化を振り返り
- プロンプトを改善
- ROIを測定
```
