## 5.1 データベース基礎

### データベースとは

**簡単に言うと**: データを整理して保存する「デジタルの本棚」

**定義**:
- 大量のデータを効率的に保存・検索・管理するシステム
- ExcelやGoogleスプレッドシートの超強力版
- Webサイトやアプリの裏側で動いている

**具体例**:
- ECサイト: 商品情報、在庫、注文履歴を保存
- SNS: ユーザー情報、投稿、いいねを保存
- 銀行: 口座情報、取引履歴を保存

**データベースの種類**:

1. **リレーショナルデータベース（RDB）**
   - データを表（テーブル）形式で管理
   - Excelのような表をイメージ
   - 代表的なもの: MySQL、PostgreSQL、Oracle
   - 用途: 一般的なWebサイト、業務システム

```
ユーザーテーブル
┌────┬────────┬─────────────┬────────┐
│ID  │ 名前   │ メール      │ 年齢   │
├────┼────────┼─────────────┼────────┤
│ 1  │ 田中   │ tanaka@...  │ 25    │
│ 2  │ 佐藤   │ sato@...    │ 30    │
└────┴────────┴─────────────┴────────┘
```

2. **NoSQLデータベース**
   - 表形式にとらわれない柔軟なデータ管理
   - JSON形式でデータを保存
   - 代表的なもの: MongoDB、DynamoDB、Redis
   - 用途: ビッグデータ、リアルタイムアプリ

**実務での会話例**:
- 「ユーザー数が100万人を超えたので、データベースをスケールアップしましょう」
- 「このクエリが遅いので、インデックスを張りましょう」
- 「データベースのバックアップは毎日取っていますか？」

**よくある質問**:
- Q: データベースとExcelの違いは？
- A: Excelは数万行までですが、データベースは数億行でも高速に処理できます。また、同時に何千人がアクセスしても大丈夫です。

### API（Application Programming Interface）

**簡単に言うと**: アプリ同士がデータをやり取りするための「窓口」

**定義**:
- システム間でデータをやり取りする仕組み
- レストランで言うと「メニュー」のようなもの
- メニュー（API）を見て注文（リクエスト）すると、料理（データ）が届く

**APIの仕組み**:

```
   あなたのアプリ          API          外部サービス
        │                  │                │
        │  天気を教えて!    │                │
        ├─────────────────>│                │
        │                  │  天気データ要求 │
        │                  ├───────────────>│
        │                  │                │
        │                  │  天気: 晴れ 25℃│
        │  晴れ 25℃       │<───────────────┤
        │<─────────────────┤                │
```

**代表的なAPI**:

1. **Google Maps API**
   - 地図表示、ルート検索、場所検索
   - 自分のWebサイトに地図を埋め込める
   - 月28,000回まで無料

2. **Twitter API**
   - ツイートの投稿、取得
   - タイムラインの表示
   - 自動投稿botなどに使用

3. **Stripe API**
   - クレジットカード決済
   - 自分のサイトに決済機能を追加

4. **LINE Messaging API**
   - LINEボットの作成
   - メッセージの送受信

**APIのメリット**:
1. **開発時間の短縮**: ゼロから作らなくて済む
2. **コスト削減**: 既存サービスを活用
3. **品質向上**: 専門企業のサービスを利用
4. **保守が楽**: 更新を相手がやってくれる

**実務での会話例**:
- 「決済機能はStripe APIで実装しましょう」
- 「Google Maps APIの無料枠を超えそうなので、料金プランを確認してください」
- 「このAPIはレート制限があるので、秒間100リクエストまでに抑えましょう」

**料金体系**:
- 無料枠あり: Google Maps、Twitter、多くのAPIで一定量まで無料
- 従量課金: 使った分だけ支払い（1,000リクエストあたり○円など）
- 月額固定: 月額数千円から数万円

**よくある質問**:
- Q: APIを使うのに許可は必要ですか？
- A: はい。各サービスでアカウント登録とAPI キーの取得が必要です。また、利用規約を守る必要があります。

## 5.2 クラウドサービスの種類

**簡単に言うと**: インターネット上のサーバーやソフトウェアを借りて使うサービス

**従来の方式との違い**:

```
従来（オンプレミス）        クラウド
┌─────────────────┐    ┌─────────────────┐
│自社でサーバー購入│    │月額で借りる      │
│初期費用: 数百万円│    │初期費用: 0円    │
│設置場所が必要   │    │設置場所不要      │
│故障したら自分で │    │故障時も自動対応  │
└─────────────────┘    └─────────────────┘
```

### IaaS（Infrastructure as a Service）

**簡単に言うと**: サーバーやネットワークなどの「インフラ」を借りるサービス

**定義**:
- インフラ（基盤）をサービスとして提供
- 仮想サーバー、ストレージ、ネットワークを借りられる
- 自由度が高いが、管理は自分で行う

**イメージ**:
```
マンションの例で例えると...

オンプレミス        IaaS            PaaS          SaaS
（一戸建て）      （スケルトン）   （内装済み）   （家具付き）
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│土地から購入│    │部屋だけ借り│    │設備も完備  │    │すぐ住める│
│全部自分で │    │内装は自分で│    │家具は自分  │    │何もいらない│
└─────────┘    └─────────┘    └─────────┘    └─────────┘
```

**代表的なIaaS**:
1. **AWS EC2**
   - 仮想サーバーをレンタル
   - 料金: 月1,000円から（小規模）
   - 用途: Webサーバー、アプリサーバー

2. **Google Compute Engine**
   - Googleのインフラを利用
   - 料金: AWSより若干安い
   - 用途: データ処理、AI学習

3. **Microsoft Azure Virtual Machines**
   - Windowsとの相性が良い
   - 料金: AWSと同程度
   - 用途: 企業システム

**メリット**:
- 自由度が高い（OSから選べる）
- スケーラブル（規模を柔軟に変更）
- 初期費用不要

**デメリット**:
- サーバー管理が必要（セキュリティ更新など）
- 専門知識が必要
- 運用コストがかかる

**実務での会話例**:
- 「IaaSでサーバーを立てて、自由にカスタマイズしましょう」
- 「EC2インスタンスのスペックを上げて、処理速度を改善します」

### PaaS（Platform as a Service）

**簡単に言うと**: アプリを動かす「プラットフォーム」を借りるサービス

**定義**:
- アプリ開発・実行環境を提供
- サーバー管理不要、コードを書くだけ
- OSやミドルウェアは自動管理

**IaaSとの違い**:
```
IaaS                        PaaS
┌─────────────────┐    ┌─────────────────┐
│アプリ（自分）    │    │アプリ（自分）    │
│OS（自分で管理）  │    │自動管理される    │
│サーバー（借りる）│    │サーバー（借りる）│
└─────────────────┘    └─────────────────┘
```

**代表的なPaaS**:
1. **Heroku**
   - コードをpushするだけでデプロイ
   - 料金: 月1,000円から
   - 用途: Webアプリ、スタートアップ

2. **Google App Engine**
   - 自動スケーリング
   - 料金: 従量課金
   - 用途: トラフィックが変動するアプリ

3. **AWS Elastic Beanstalk**
   - AWSのPaaSサービス
   - 料金: 使った分だけ
   - 用途: Webアプリケーション

**メリット**:
- サーバー管理不要
- 開発に集中できる
- 自動スケーリング

**デメリット**:
- カスタマイズに制限
- 特定プラットフォームに依存
- IaaSより高コストになることも

**実務での会話例**:
- 「PaaSを使えば、サーバー管理不要で開発に集中できます」
- 「Herokuにデプロイしたので、すぐに公開できます」

### SaaS（Software as a Service）

**簡単に言うと**: ソフトウェアをインターネット経由で使うサービス

**定義**:
- 完成したソフトウェアをブラウザで利用
- インストール不要、アカウント登録だけ
- 最もシンプルで使いやすい

**身近なSaaS例**:
```
業務別のSaaS例

ビジネス一般
- Gmail（メール）
- Google ドライブ（ファイル共有）
- Slack（チャット）
- Zoom（ビデオ会議）

営業・マーケティング
- Salesforce（顧客管理）
- HubSpot（マーケティング）
- Mailchimp（メール配信）

経理・人事
- freee（会計ソフト）
- マネーフォワード（経費精算）
- SmartHR（人事労務）

開発・デザイン
- GitHub（コード管理）
- Figma（デザインツール）
- Notion（ドキュメント管理）
```

**代表的なSaaS**:
1. **Salesforce**
   - 顧客管理（CRM）
   - 料金: 月3,000円/ユーザー
   - 用途: 営業管理、顧客情報管理

2. **Google Workspace**
   - Gmail、ドライブ、ドキュメント
   - 料金: 月680円/ユーザー
   - 用途: メール、ファイル共有

3. **Slack**
   - ビジネスチャット
   - 料金: 月850円/ユーザー
   - 用途: 社内コミュニケーション

4. **freee**
   - クラウド会計ソフト
   - 料金: 月1,980円から
   - 用途: 経理、確定申告

**メリット**:
- すぐに使える（インストール不要）
- 自動アップデート
- どこからでもアクセス
- 初期費用が安い

**デメリット**:
- カスタマイズできない
- データが外部にある
- 月額料金が継続的にかかる
- サービス終了のリスク

**実務での会話例**:
- 「Slackを導入して、メールを減らしましょう」
- 「freeeなら、経理の自動化ができます」
- 「SalesforceでCVRが30%改善しました」

### IaaS vs PaaS vs SaaS 比較表

```
┌──────────┬──────────┬──────────┬──────────┐
│項目      │ IaaS     │ PaaS     │ SaaS     │
├──────────┼──────────┼──────────┼──────────┤
│自由度    │高        │中        │低        │
│管理の手間│大        │中        │小        │
│初期費用  │低        │低        │極小      │
│専門知識  │必要      │やや必要  │不要      │
│例        │AWS EC2   │Heroku    │Slack     │
└──────────┴──────────┴──────────┴──────────┘
```

**どれを選ぶべきか？**

```
選択フロー

既製品で十分？
  ├─ YES → SaaS（例: Slack、freee）
  └─ NO
      ↓
  独自開発が必要？
      ├─ YES
      │    ↓
      │  細かい制御が必要？
      │    ├─ YES → IaaS（例: AWS EC2）
      │    └─ NO → PaaS（例: Heroku）
      │
      └─ NO → SaaS
```

**実務での会話例**:
- 「まずはSaaSで試して、足りなければPaaSで開発しましょう」
- 「IaaSは管理コストが高いので、PaaSから始めましょう」
- 「SaaSで月10万円かかるなら、独自開発も検討しましょう」

**料金比較の例**:

```
チャットツールの場合

SaaS（Slack）
- 月850円 × 100人 = 月85,000円
- 初期費用: 0円
- 管理不要

独自開発（IaaS）
- 開発費: 500万円
- サーバー代: 月5万円
- 保守費用: 月10万円
→ 3年で使うなら: 500万 + 15万×36ヶ月 = 1,040万円
→ SaaSの方が安い（85,000円×36ヶ月 = 306万円）
```

**よくある質問**:
- Q: SaaSとアプリの違いは？
- A: SaaSはブラウザで使うソフトウェア、アプリはスマホにインストールするソフトウェアです。ただし、最近は両方提供するサービスも多いです。
