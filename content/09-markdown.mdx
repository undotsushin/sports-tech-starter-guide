## 9.1 マークダウンとは

**簡単に言うと**: 文章を簡単に装飾できる記法

**定義**:
- プレーンテキストで書ける
- HTMLに変換できる
- 読みやすく、書きやすい

**マークダウンが使われている場所**:
```
よく使う場所
- GitHub（コード管理）
- Notion（ドキュメント管理）
- Slack（チャット）
- Discord（コミュニケーション）
- Reddit（掲示板）
- Qiita（技術ブログ）
- Zenn（技術記事）
- はてなブログ
- AI（ChatGPT、Claudeなど）
```

**なぜマークダウンが人気なのか**:
```
1. シンプル
   - 覚えることが少ない
   - 5分で基本を習得

2. 読みやすい
   - プレーンテキストでも読める
   - 装飾がシンプル

3. どこでも使える
   - テキストエディタで書ける
   - 様々なサービスで対応

4. バージョン管理しやすい
   - Gitで差分が見やすい
   - 複数人での編集が楽

5. AIとの相性が良い
   - ChatGPTやClaudeが理解しやすい
   - プロンプトで「マークダウンで」と指定できる
```

## 9.2 基本記法

### 見出し

**簡単に言うと**: 文章の階層を表現する

```markdown
# 大見出し（H1）- 文書のタイトル
## 中見出し（H2）- 章
### 小見出し（H3）- 節
#### より小さい見出し（H4）
##### さらに小さい見出し（H5）
###### 最小の見出し（H6）
```

**実際の表示**:
```
# プロジェクト概要
## 背景
### 市場環境
#### ターゲット市場
```

**使い分け**:
```
H1: 文書全体のタイトル（1回だけ）
H2: 大きな章立て
H3: 章の中の節
H4以下: 必要に応じて使う（あまり使わない）
```

### 文字装飾

**太字**:
```markdown
**太字**
__太字__（アンダースコア2つでも可）
```

**斜体**:
```markdown
*斜体*
_斜体_（アンダースコア1つでも可）
```

**太字 + 斜体**:
```markdown
***太字かつ斜体***
```

**取り消し線**:
```markdown
~~取り消し線~~
```

**インラインコード**:
```markdown
`code`
```

**実際の使用例**:
```markdown
この機能は**重要**です。
詳細は*後述*します。
~~この機能は廃止されました~~
`npm install`コマンドを実行してください。
```

### リスト

**順序なしリスト**:
```markdown
- 項目1
- 項目2
- 項目3
  - 入れ子項目1
  - 入れ子項目2
```

**順序ありリスト**:
```markdown
1. 手順1
2. 手順2
3. 手順3
   1. 詳細手順1
   2. 詳細手順2
```

**チェックボックス（タスクリスト）**:
```markdown
- [ ] 未完了タスク
- [x] 完了タスク
- [ ] 進行中タスク
```

**実際の使用例**:
```markdown
## 今週のタスク
- [x] 提案書作成
- [x] クライアント訪問
- [ ] 見積書送付
- [ ] 契約書確認
```

### リンク

**基本的なリンク**:
```markdown
[リンクテキスト](URL)
```

**実例**:
```markdown
[Google](https://google.com)
[GitHub](https://github.com)
```

**参考リンク形式**:
```markdown
[リンク][参照名]

[参照名]: URL "タイトル"
```

**実例**:
```markdown
詳細は[公式ドキュメント][docs]を参照してください。

[docs]: https://example.com "公式ドキュメント"
```

### 画像

**基本的な画像**:
```markdown
![代替テキスト](画像URL)
```

**実例**:
```markdown
![ロゴ](https://example.com/logo.png)
```

**サイズ指定（HTML混在）**:
```markdown
<img src="画像URL" width="300">
```

### 引用

**引用ブロック**:
```markdown
> これは引用です
> 複数行も可能です
```

**入れ子の引用**:
```markdown
> 引用レベル1
>> 引用レベル2
>>> 引用レベル3
```

**実際の使用例**:
```markdown
> クライアントからのフィードバック:
> 「デザインは良いが、読み込みが遅い」
```

### 水平線

```markdown
---
***
___
```

**使い方**:
```markdown
## セクション1
内容...

---

## セクション2
内容...
```

## 9.3 表とコードブロック

| 列1 | 列2 | 列3 |
|-----|-----|-----|
| A   | B   | C   |
| D   | E   | F   |

**列の配置**:
```markdown
| 左寄せ | 中央 | 右寄せ |
|:-------|:----:|-------:|
| Left   | Center | Right |

**実務での使用例**:

### 仕様書

```markdown
# ログイン機能仕様書

## 概要
ユーザーがメールアドレスとパスワードでログインできる機能

## 要件
### 機能要件
1. メールアドレスとパスワードで認証
2. ログイン失敗は3回まで
3. ログイン後はダッシュボードに遷移

### 非機能要件
- レスポンスタイム: 2秒以内
- 同時ログイン: 10,000人まで対応

## API仕様
### エンドポイント
```
POST /api/auth/login
```

### リクエスト
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

### レスポンス（成功）
```json
{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "name": "田中太郎"
  }
}
```

### レスポンス（失敗）
```json
{
  "success": false,
  "error": "メールアドレスまたはパスワードが間違っています"
}
```

## エラーハンドリング
| エラーコード | メッセージ | 対応 |
|-------------|-----------|------|
| 401 | 認証エラー | 再入力を促す |
| 429 | ログイン制限 | 30分後に再試行 |
| 500 | サーバーエラー | 管理者に通知 |

## セキュリティ
> **重要**: パスワードは必ずbcryptで暗号化すること

- パスワードはbcryptで暗号化
- JWTトークンは1時間で期限切れ
- HTTPS必須

## テストケース
- [ ] 正常なログイン
- [ ] メールアドレス間違い
- [ ] パスワード間違い
- [ ] 3回失敗後のロック
- [ ] トークンの有効期限切れ
```

## 9.6 よくある間違いと対処法

**間違い1: リストの前に空行がない**
```markdown
× 悪い例:
文章があって
- リスト1
- リスト2

○ 良い例:
文章があって

- リスト1
- リスト2
```

**間違い2: 見出しの前後に空行がない**
```markdown
× 悪い例:
文章があって
## 見出し
次の文章

○ 良い例:
文章があって

## 見出し

次の文章
```

**間違い3: コードブロックの言語指定がない**
```markdown
× 悪い例:
```
console.log("hello");
```

○ 良い例:
```javascript
console.log("hello");
```
```

**間違い4: 表の区切り線が不揃い**
```markdown
× 悪い例:
| A | B |
|--|--|
| 1 | 2 |

○ 良い例:
| A | B |
|---|---|
| 1 | 2 |
```
